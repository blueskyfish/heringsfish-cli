Plugin Overview

A plugin is a javascript module and executes an action. The name of the plugin is the name of the action too.

Builtin Plugins

An overview of the builtin plugins.

Plugin Action Description
Help help Shows the help to the action / plugin
Init init Initial the Heringsfish server config in the current project.
Application Server  
Create Domain create-domain Creates the domain on the application server.
Remove Domain remove-domain Remove and delete the domain on the application server.
Start Server start-server Starts the application server with the domain.
Stop Server stop-server Stops the application server with the domain.
Restart Server restart-server Stops and starts the application server with the domain.
Maven  
Build Maven build-maven Build the application with maven.
Clean Maven clean-maven Executes the maven goal clean.
Test Maven test-maven Executes the maven goal test.
Deploy Maven deploy-maven Build the application with maven nd deploy on the application server.
Derby Database  
Start Derby Database start-derby Starts the builtin Derby database on the application server.
Stop Derby Database start-derby Stops the builtin Derby database on the application server.

Create an own Plugin

This section describes how to build your own plugin for heringsfish.

  • Defines a javascript node module.
  • The module exports only one function that receive one parameter from type Options.

Example

/*!
 * Your module
 */

/**
 * This is the only function of the module exports
 * 
 * @param {Options} options contains some functionality of the heringsfish
 * @return {Promise<*>}
 */
module.exports = function (options) {
  return new Promise(function(resolve, reject) {

    // reject({
    //   code: 0xffffff,
    //   message: 'Error',
    // });
    resolve({
      message: 'Success'
    });
  });
}
